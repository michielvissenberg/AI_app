POST-MVP (architecture hardening)

1) Provenance fields (add to statement + line-item outputs)
- run_id: unique UUID per pipeline execution for cross-stage traceability.
- source_pdf_path + source_pdf_sha256: immutable source identity and dedupe key.
- source_engine_primary / source_engine_effective: selected engine vs actual engine used after fallback.
- extraction_started_at / extraction_completed_at: stage timing for SLA + incident analysis.
- pipeline_version + mapper_rules_version + evaluation_benchmark_version: reproducibility across releases.
- confidence fields: statement_type_confidence, period_alignment_confidence, parse_confidence per line item.
- lineage pointers: source_table_id, source_page_range, source_row_index, source_column_indexes.

2) Hybrid fallback strategy (engine + quality-gate routing)
- Primary path: Docling (faster/local control), Azure secondary for recoverability.
- Trigger fallback when any gate fails:
  a) extraction gate: table_cells == 0 or stage error classified recoverable,
  b) mapping gate: statement_detection_accuracy < 95,
  c) parsing gate: numeric_parse_accuracy < 98,
  d) schema gate: validation failure or missing critical anchors (revenue/net_income/total_assets).
- Fallback modes:
  Mode A: full rerun with alternate engine,
  Mode B: selective rerun on failed page ranges only,
  Mode C: merge strategy preferring higher-confidence row blocks.
- Persist both attempts with provenance and mark final_resolution = primary|fallback|merged.

3) Persistence layer rollout (after MVP accuracy stabilization)
- Stabilization criteria before write-enable:
  - statement_detection_accuracy >= 97 for 10 consecutive sample runs,
  - line_item_mapping_accuracy >= 97 for 10 consecutive sample runs,
  - numeric_parse_accuracy >= 99 for 10 consecutive sample runs.
- Collection design (MongoDB):
  - filings_raw: immutable run outputs + provenance,
  - filings_curated: latest accepted statement per (ticker, period_ending, report_type),
  - evaluations: KPI reports + mismatch diagnostics per run.
- Keys + indexes:
  - unique(ticker, period_ending, report_type, source_pdf_sha256, pipeline_version),
  - index(run_id), index(extracted_at), index(source_engine_effective), index(kpi buckets).
- Write policy:
  - upsert curated only if new run passes KPI thresholds and validation,
  - otherwise store as raw/evaluation only and raise review_required flag.
